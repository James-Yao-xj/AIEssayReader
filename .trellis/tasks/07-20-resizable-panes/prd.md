# 三栏可拖拽调整宽度

## Goal

三栏布局（左 PDF / 中文本 / 右 AI）的列宽支持鼠标拖拽分隔条来调整，替代当前固定的 flex 比例。

## Context

当前 `.pane--pdf` / `.pane--text` / `.pane--ai` 使用固定 flex 值（1.2 / 1 / 1.1），用户无法按需调整各栏宽度。读论文时可能需要 PDF 宽一点看图表，或 AI 面板宽一点读分析结果，固定比例不够灵活。

## Requirements

- **R1 分隔条**：在 PDF-文本、文本-AI 之间各插入一条可拖拽的分隔条（gutter），视觉上约 4~6px 宽，hover 时变色 + 光标变为 `col-resize`。
- **R2 拖拽调整**：按住分隔条水平拖动时，实时调整相邻两栏的宽度（左边栏增减、右边栏反向增减，总宽不变）。
- **R3 最小宽度**：每栏最小宽度 200px（或约 15%），拖拽到临界值时停止不再缩小。
- **R4 视觉反馈**：拖拽中分隔条高亮（颜色加深），光标保持 `col-resize`；可在 body 上设 `user-select: none` 防文字选中干扰。
- **R5 宽度持久化**：用户调整后的比例保存到 localStorage（如 `aie:pane-ratios`），刷新页面后恢复。
- **R6 响应式**：窄屏（≤960px）三栏堆叠时，分隔条隐藏，拖拽功能禁用。
- **R7 双击重置**：双击分隔条恢复该分隔条两侧栏为默认比例（1:1:1 均分或原始 1.2:1:1.1）。

## Acceptance Criteria

- [ ] 两分隔条在宽屏正常显示，hover 变色 + 光标变化。
- [ ] 拖拽分隔条可实时调整相邻栏宽度，第三栏不受影响。
- [ ] 任一侧拖到 200px 以下时停止缩小。
- [ ] 刷新页面后宽度比例保持。
- [ ] 窄屏（≤960px）分隔条不可见、不可拖拽。
- [ ] 双击分隔条恢复默认比例。
- [ ] `npm run build` 成功产出单 HTML。

## Out of Scope

- 垂直方向的高度调整（仅水平拖拽）。
- 拖拽动画/过渡效果。
- 双击重置为上一次手动设置的比例（仅重置为系统默认）。

## Notes

- 纯 vanilla JS 实现，不引入第三方拖拽库。
- 分隔条作为独立 DOM 元素插入 `.app-main` 中，位于 pane 之间。
- 默认比例沿用现有的 1.2:1:1.1（PDF 稍宽），可作为"恢复默认"的目标值。
