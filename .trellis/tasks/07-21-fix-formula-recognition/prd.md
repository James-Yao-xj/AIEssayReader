# 修复PDF公式识别乱码并支持LaTeX渲染

## Goal

双重方案解决公式识别问题：
1. **快速模式**（已有）：pdf.js 文本提取，字号感知算法
2. **AI 精确识别**（新增）：用 vision 模型识别页面图片，直接输出含 LaTeX 的结构化文本

用户可自选模式。

## Requirements

1. **保留 pdf.js 快速提取**：当前 `extractText()` 作为默认快速模式
2. **新增 AI 视觉识别**：PDF 页面渲染为图片 → 发送给 vision 模型 → 返回含 LaTeX 的文本
3. **中栏按钮**：在文本中栏头部添加"AI 识别"按钮，点击触发全页识别
4. **进度反馈**：识别过程显示进度（"识别中 3/10…"），支持取消
5. **结果替换**：AI 识别完成后替换中栏文本，同时更新 `store.paper` 供 AI 分析使用
6. **复用已有管线**：OpenAI 兼容 API 通道 + KaTeX 渲染全部复用

## Acceptance Criteria

- [ ] pdf.js 快速提取仍正常工作（默认模式）
- [ ] 点击"AI 识别"按钮后，逐页发送给 vision 模型识别
- [ ] 识别结果中 LaTeX 公式被中栏 KaTeX 正确渲染
- [ ] 进度显示（当前页/总页数），支持中途取消
- [ ] 识别结果写入 store.paper，AI 分析（总结/概念/质疑）能引用正确文本
- [ ] API 未配置时按钮给出明确提示
- [ ] `npm run build` 构建成功

## Out of Scope

- 扫描件直接识别（天然支持，但不单独强调）
- 批量多文件
- 结果缓存/持久化

## Decisions

| 决策 | 选择 |
|------|------|
| 模式关系 | 快速模式（pdf.js）默认 + AI 识别按钮备选 |
| Vision API | 复用现有 OpenAI 兼容通道 |
| 页面渲染 | 复用 pdf.js canvas 渲染，scale=2.0 |
| 并发 | 逐页串行（避免 API 限流），支持取消 |
