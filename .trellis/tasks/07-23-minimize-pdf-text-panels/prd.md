# 原文PDF与文字提取版面最小化

## Goal

在「原文 PDF」（左栏 `#pane-pdf`）和「文字提取」（中栏 `#pane-text`）两个版面各提供一个「－」（最小化）按钮。点击后，该版面收成一条窄竖条（侧边细条），其余版面自动加宽填满被释放的空间；点击该细条即可恢复原版面。右栏 AI 面板不在本次范围内。

## Background

当前布局是 `.app-main` 内三栏水平 flex（PDF / 文字提取 / AI），栏宽由 `src/ui/paneResize.js` 用行内 `style.width` 百分比控制，比例持久化在 `localStorage` 的 `aie:pane-ratios`。三个 pane 目前都没有标题栏（只有空态占位 + 滚动区）。窄屏（≤960px）时三栏自动纵向堆叠。

## Requirements

### 功能需求

- R1 「原文 PDF」和「文字提取」两栏各自有最小化入口（「－」按钮）。
- R2 点击「－」后，目标栏收成一条窄竖条（固定宽度，约 36px），其内容（空态/滚动区）隐藏，仅保留一条可识别的竖条（含版面名称与恢复提示）。
- R3 栏被最小化时，其余未被最小化的栏按各自原有比例自动加宽，填满释放的空间。
- R4 点击最小化后的竖条（或竖条上的按钮）可恢复该栏到最小化前的宽度比例。
- R5 两栏可同时最小化（此时仅 AI 栏占满）。
- R6 最小化状态不持久化：刷新页面后，所有栏恢复为正常的展开三栏（沿用 `aie:pane-ratios` 的比例）。

### 约束

- C1 右栏 AI 面板不增加最小化按钮，不在本次范围。
- C2 最小化仅在宽屏（横向三栏）布局下生效；窄屏（≤960px，纵向堆叠）下隐藏「－」按钮、最小化为 no-op。
- C3 最小化期间，拖拽分隔条（gutter）调整栏宽的功能暂停（避免与固定宽度竖条的比例计算冲突），全部恢复后自动恢复拖拽。
- C4 不破坏现有交互：点击 PDF 空态选文件、文本选中追问、AI 面板等既有行为不受影响。
- C5 沿用项目现有约定：原生 ES Modules、BEM 类名、单 `styles.css`、`[data-theme="dark"]` 暗色覆盖、JSDoc 注释、不引入新依赖。
