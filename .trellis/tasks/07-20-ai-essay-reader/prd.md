# AI 论文阅读插件

## Goal

做一个**单 HTML 形态**的 AI 论文阅读插件。用户把 PDF 论文拖进网页，插件调用 AI 帮忙：总结论文、解释关键概念、对论文提出质疑，并支持基于该论文的对话问答。

解决的核心痛点（来自 README）：论文越来越多、读起来眼花；英文阅读疲劳；打印成本高。

## Confirmed Facts（已确认）

- **最终交付物形态**：单个自包含 `dist/index.html`（CSS/JS/PDF worker 全部内联），双击即用、可离线分发。
- **工程结构**：Vite 工程化开发（`src/` 分模块），`vite-plugin-singlefile` 打包成单 HTML。
- **AI 后端**：用户自填 `API Key` + `Base URL`，采用 **OpenAI 兼容协议**（可对接 OpenAI / DeepSeek / OpenRouter / 本地 vLLM / Ollama OpenAI 接口 等）。做一个可切换的 provider 抽象层。Anthropic 原生 messages API 留作 provider 扩展点，MVP 不做。
- **核心功能**：
  1. 拖拽 PDF 上传
  2. 总结论文
  3. 解释关键概念
  4. 提出质疑（批判性分析）
  5. 基于论文的对话问答
- **布局（已确认）**：三栏 —— 左 PDF 原版渲染 / 中 提取的可选中文本 / 右 AI 面板（总结·概念·质疑·对话）。信息密度最高，适合深度研读。
- **公式渲染（已确认）**：KaTeX，**仅挂右栏 AI 输出**（中栏 PDF 提取的文本不是干净 LaTeX，渲染无意义）；字体惰性加载（首次用到才 fetch）、流式输出时节流渲染（避免每 token 全量重渲染）。pdf.js 已是体积大头，KaTeX 增量可接受。
- **AI 上下文策略（已确认）**：C —— 论文全文作为持久 system context 注入一次，"总结/概念/质疑/对话"都在同一会话追加；对话历史过长时**滑窗裁剪**（保留 system + 论文全文 + 最近 N 轮对话）。不做分段检索（留作后期）。
- **提示词（已确认）**：总结/概念/质疑/对话 四个任务先用内置高质量中文模板，后期再加设置面板让用户改。
- **输出风格约束（已确认，平台级强制）**：① 非必要不使用比喻句，使用时须再三考量比方是否恰当、是否会误导；② 解释专业术语时同时给出**官方专业定义** + **生动简明扼要的通俗解释**。作为 `GLOBAL_STYLE` 前置于所有任务的 system prompt，不受用户自定义任务模板影响。
- **当前仓库状态**：仅 `README.md`（含"问题描述"），其余为空；`.trellis/spec/` 是 `trellis init` 的 Electron/React/TS 模板默认值，**与本项目实际技术栈不符**，设计时不应盲从。

## Requirements

- **R1 文件输入**：拖拽 PDF 到页面，也支持点击选择文件；处理中显示 loading，错误（非 PDF / 解析失败）给出明确提示。
- **R2 PDF 原版渲染（左栏）**：用 pdf.js 渲染原版页面，支持滚动浏览；这是用户核实 AI 说法、看图表/公式原版的来源。
- **R3 PDF 文本提取（中栏）**：浏览器端提取全文文本，按页/分段保留结构，可选中、可复制；选中文本可一键"追问"送入对话。
- **R4 设置面板**：填写 Base URL / API Key / 模型名 / 温度等，本地持久化（localStorage），Key 不回显明文；未填时引导填写。
- **R5 三大分析**：一键"总结 / 概念解释 / 批判质疑"，流式输出，Markdown + KaTeX 渲染到右栏对应 tab/区。所有产出须符合 `GLOBAL_STYLE`（慎用比喻、术语双重解释）。
- **R6 论文对话**：右栏对话区，针对当前论文多轮问答；上下文采用全文注入 + 滑窗裁剪。
- **R7 离线与单 HTML**：纯前端，`npm run build` 产出单个自包含 HTML；除调用 AI 接口的网络请求外无需联网或服务器。
- **R8 健壮性**：未填 Key、网络错误、流式中断、超长论文等情况均有明确反馈，不静默失败。

## Acceptance Criteria

- [ ] `npm run build` 产出单个 `dist/index.html`，断网双击在浏览器打开仍能加载（拖入本地 PDF、查看文本提取均可用）。
- [ ] 拖入一篇真实 PDF 后：左栏渲染原版、中栏展示提取文本（可选中复制）。
- [ ] 填好 Base URL + Key + 模型后，"总结/概念/质疑"任一按钮可流式输出，公式正确渲染。
- [ ] 对话区能基于论文内容多轮问答，且连续多轮后不因上下文超限报错（滑窗生效）。
- [ ] 中栏选中文本可一键送入对话追问。
- [ ] 设置项刷新页面后仍保留；未填 Key 时点击分析按钮给出明确提示。
- [ ] 流式中断 / 网络错误有可恢复的提示与重试入口。
- [ ] 概念解释类输出同时包含官方定义与通俗解释；全文无不当比喻（`GLOBAL_STYLE` 生效）。

## Out of Scope

- 多篇论文库 / 跨论文检索（先做单篇）。
- 分段摘要 + 向量检索式 RAG（留作后期优化）。
- Anthropic 原生 messages API（留作 provider 扩展点）。
- 用户自定义提示词编辑器（先内置模板）。
- 用户登录、云同步、服务器端组件。
- PDF 编辑 / 标注 / 导出。

## Open Questions

- 暂无阻塞规划的问题；功能细节（如"质疑"的侧重、概念解释的粒度、对话的默认行为）将在用户后续描述功能时细化并回写本文件。
