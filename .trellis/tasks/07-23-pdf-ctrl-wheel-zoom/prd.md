# PDF Ctrl+滚轮独立缩放

## Goal

当鼠标悬浮在左栏「原文 PDF」的页面上时，按住 `Ctrl`（Mac 为 `Cmd`）滚动滚轮，只缩放 PDF 本身，不再触发浏览器的整页缩放。缩放只作用于 PDF 栏，不影响中栏文本、右栏 AI 面板或窗口其它部分。

## Background

- PDF 原版渲染在 `src/pdf/render.js`：把每页用 pdf.js 渲染成 `<canvas>`，插入 `#pdf-scroll` 容器（`.pane--pdf .pane__scroll`），用 `IntersectionObserver` 懒加载（400px 提前量），渲染倍率固定 `scale = 1.5`。
- 当前 canvas 的 CSS 是 `.pane--pdf canvas { max-width: 100%; height: auto; }`，即「适应栏宽」：渲染出的位图被 CSS 等比缩放到栏宽显示。
- 目前没有任何 PDF 缩放逻辑，`Ctrl+滚轮` 走浏览器默认行为（缩放整个网页）。

## Requirements

### 功能需求

- R1 鼠标悬浮在 PDF 页面上时，`Ctrl+滚轮`（或 `Cmd+滚轮`）缩放 PDF 内容，且 `preventDefault` 阻止浏览器整页缩放。
- R2 缩放只影响 PDF 栏；中栏文本、右栏 AI 面板、顶栏、窗口尺寸均不受影响。
- R3 缩放是「清晰重渲染」：放大后文字保持锐利（按新倍率重绘 canvas 位图，而非 CSS 拉伸模糊）。
- R4 缩放以光标所在位置为锚点：放大/缩小过程中，光标指向的页面内容尽量停留在光标下方（接近真实 PDF 阅读器的「向光标处缩放」体验）。
- R5 缩放超出栏宽时出现滚动条，可横向/纵向滚动查看（不被裁切隐藏）。
- R6 不按 `Ctrl` 的普通滚轮仍按现状纵向滚动 PDF（行为不变）。
- R7 触控板双指捏合（浏览器合成 `ctrlKey` 的 wheel 事件）同样触发 PDF 缩放。

### 约束

- C1 缩放范围有上下限，避免过大导致 canvas 位图爆炸（内存）/ 过小无意义。初始（userZoom=1）等价于当前的「适应栏宽」。
- C2 缩放状态不持久化：切换/重新加载 PDF 时重置为 1（适应栏宽）。刷新页面后回到默认（与现有最小化状态「不持久化」的取向一致）。
- C3 不破坏现有交互：PDF 拖入/渲染、文本提取、选中追问、AI 面板、版面最小化/拖拽、设置、主题切换均不受影响。
- C4 不按 `Ctrl` 时绝不 `preventDefault`（保证普通滚动与页面可访问性）。
- C5 窄屏（≤960px，纵向堆叠）下缩放同样可用（PDF 栏仍可滚动）。
- C6 沿用项目约定：原生 ES Modules、BEM 类名、单 `styles.css`、`[data-theme="dark"]` 暗色覆盖、JSDoc 注释、不引入新依赖、模块单向依赖（`pdf/` 不依赖 `ui/`）。
- C7 缩放逻辑封装在 `pdf/render.js` 内部（事件绑定 + 清理一并随 `renderPdf` 生命周期），`main.js` 仅消费返回的 handle，不感知缩放实现细节。

## Acceptance Criteria

- [ ] AC1 鼠标悬停在 PDF 页面上 `Ctrl+滚轮` 上滚 → PDF 放大；下滚 → 缩小；且整页不被浏览器缩放（地址栏旁的缩放比例不变）。
- [ ] AC2 缩放期间，中栏文本字号/布局、右栏 AI 面板、顶栏、窗口大小均无变化。
- [ ] AC3 放大到文字仍清晰锐利（位图按新倍率重绘，非 CSS 拉伸发虚）。
- [ ] AC4 连续 `Ctrl+滚轮` 缩放时，光标指向的内容大体停留在光标下方（无明显跳屏到顶部/其它页）。
- [ ] AC5 放大超过栏宽后，PDF 栏出现横向滚动条可左右拖动查看；纵向同理可滚动。
- [ ] AC6 不按 `Ctrl` 仅滚动滚轮 → PDF 正常上下滚动（与现状一致），整页不缩放。
- [ ] AC7 缩放到上下限时继续同向滚动无异常（无报错、不溢出）。
- [ ] AC8 切换/重新拖入一个新 PDF 后，缩放重置为默认（适应栏宽）。
- [ ] AC9 最小化 PDF 栏后再恢复、拖拽调整栏宽、暗色模式切换，缩放功能均仍可用、无破样。
- [ ] AC10 窄屏（≤960px）下 `Ctrl+滚轮` 缩放 PDF 仍生效。
- [ ] AC11 既有功能回归正常：拖入渲染、文本提取、选中追问、AI 面板、设置、主题、版面最小化/拖拽均不受影响。
- [ ] AC12 `npm run build` 通过，无控制台报错。

## Notes

- 关键技术难点（design.md 详述）：canvas 现有 `max-width:100%` 会把放大后的位图再压缩回栏宽，使「重渲染放大」失效。需改为以「适应栏宽」为 1× 基准、按 userZoom 重渲染并显式设定显示宽度（解除 max-width 上限）。
- 锚点策略：重渲染可见页后，按光标在锚定页内的比例 + 锚定页重排后的真实 offsetTop 重算 scrollTop，保证光标处内容稳定。
- 非可见页不即时重渲染，沿用懒加载：滚动进入视口时由 IntersectionObserver 按当前倍率补渲染。
